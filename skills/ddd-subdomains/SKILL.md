# Subdomains（子域分类）

## 做什么

回答一个投资问题：哪块业务能力值得倾注最强的人和最精的模型（核心域），哪块够用就行（支撑域），哪块直接买现成（通用域）。

## 需要什么参数

- **必需**：一份含 {事件流, 泳道切分, 价值主张} 的工件。
- **可选**：竞争分析、组织战略目标、可采购的成熟方案清单。

## 怎么做

1. **抽业务能力清单**：把每个泳道/职责块提炼成一项能力（动宾短语）。
2. **逐项三分类**：核心（承载价值主张、差异化、买不到）/ 支撑（必需非差异化）/ 通用（人人一样、有成熟方案）。
3. **两条检验校准**：①做到行业最好能带来竞争优势吗？→核心；②市面有现成够用的吗？→通用。
4. **显式声明核心域**：只能少数几个；超过 2–3 个多半价值主张没想清，需收敛。
5. **标买/建/定制策略**：给每个子域一个落地策略。
6. **与价值主张对齐**：核心域必须一一呼应价值主张。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 业务能力清单 | 能力名（动宾）+ 一句话职责 + 来源泳道 |
| 子域分类表 | 子域、类型（Core/Supporting/Generic）、理由、买建策略 |
| 核心域声明 | 1–3 个核心域 + 对应价值主张条目 |
| 投入建议 | 子域 → 投入等级 → 人/模型精度预期 |

> **返回格式自检**：每项能力被唯一归类、无"既核心又通用"；核心域 ≤ 3 且各映射一条价值主张；每个通用域给出可采购方向；分类理由用业务语言（非"因为复杂"）。
