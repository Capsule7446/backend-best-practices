# Discover — 走查示例

> 附加文件，按需读取。

```text
输入：会议室预订的问题陈述 + 术语种子。

领域事件流（时间序）：
  [发起预订] —员工→ 预订已提交
  →（策略：时段冲突？）→ 冲突已检出 / 时段已锁定
  → 预订已确认（普通房）| 预订待审批（高价值房）
  → 审批已通过 / 审批已驳回 → 预订已生效 →（会议日）→ 预订已结束
  旁路：预订已取消 → 时段已释放

策略清单：
  预订已提交 → 若房为高价值 → 需审批 → 预订待审批
  预订已提交 → 若时段已占 → 冲突已检出（直接拒绝）

外部依赖：登录鉴权 → LDAP（入）；确认通知 → 邮件服务（出）

热点（红）：
  跨时区时段如何统一表达？（在"时段已锁定"，阻塞）
  审批中能否被他人抢占时段？（在"预订待审批"，重要）

泳道切分：[预订核心][审批][通知][身份]
```
