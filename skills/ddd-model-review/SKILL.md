# Model Review（模型质量评估）

## 做什么

用量化标准检查模型是否一致、完整、可落地，产出**客观评分 + 发现清单**。它只做裁判、给证据，**不包含处置建议、重跑目标或流程决策**。

## 需要什么参数

- **必需**：战略工件（子域分类、上下文目录与映射）、战术工件（不变量表、聚合目录、接口契约）。
- **可选**：范围工件的目标/非目标与价值主张（用于完整性核对）。

## 怎么做

1. **一致性检查**：通用语言是否跨工件一致？同名术语在同一上下文是否唯一？命名是否与词汇表对齐？
2. **完整性检查**：每条目标是否都有"事件流→聚合→契约"链路覆盖？有无价值主张相关路径断点。
3. **不变量健壮性**：计算**不变量表达率** = 被聚合根方法显式强制的不变量数 / 不变量总数；逐条核对归属。
4. **聚合健康**：是否有"无不变量的聚合"（数据袋）、"跨聚合强一致写"、"事件携带他聚合私有数据"。
5. **可实现性**：端口契约是否语言中立、可接口表达？仓储是否以聚合为单位？依赖是否只向内？
6. **量化打分**：五维各打分 + 总评（通过 / 有条件通过 / 不通过）。
7. **产出发现清单**：把每个未达标项写成"客观事实 + 证据 + 所在工件"，**不写处置建议、重跑目标或流程决策**。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 五维评分表 | 一致性/完整性/不变量表达率/聚合健康/可实现性，各打分 + 依据 |
| 问题清单 | 问题、严重度（阻断/重要/建议）、所在工件、证据 |
| 发现清单 | 客观事实 + 证据，不含处置建议 |
| 结论 | 通过 / 有条件通过（列条件）/ 不通过 |

> **返回格式自检**：五维都给了分与依据；每个问题都带严重度、所在工件与证据；发现是**客观事实**而非处置建议；给出了明确结论。
