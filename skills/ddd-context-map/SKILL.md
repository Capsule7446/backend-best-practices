# Context Map（上下文映射）

## 做什么

把"谁依赖谁、谁的契约说了算、用什么模式翻译"显性化。防两类灾难：核心上下文被上游模型同化（应上防腐层）、集成靠口头约定导致契约漂移（应明确所有权）。

## 需要什么参数

- **必需**：一份含 {上下文目录, 词汇表, 跨界关系待解清单} 的工件。
- **可选**：组织权力关系、现有系统既成集成、外部系统契约。

## 怎么做

1. **列关系矩阵**：所有两两有交互的上下文填入矩阵，标方向。
2. **判定权力关系**：Customer-Supplier / Conformist / Partnership / Shared Kernel（慎用）。
3. **选集成模式**：ACL 防腐层（核心上下文对外几乎都该上）/ OHS 开放主机服务 + PL 发布语言 / 事件订阅（异步最终一致）。
4. **指派契约所有权**：每条契约（API/事件 schema）必须有唯一 owner 上下文。
5. **标一致性语义**：每条跨界交互是强一致还是最终一致。
6. **画上下文映射图**：汇总上下文、关系类型、集成模式、ACL 位置。
7. **核对核心域保护**：每个核心上下文对所有上游都有 ACL 或等价隔离。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 关系矩阵 | 上下文 × 上下文：方向 + 关系类型 |
| 集成模式表 | 关系、模式、同步/异步、一致性语义 |
| 契约所有权表 | 契约、owner、消费方、版本策略 |
| 上下文映射图 | 节点=上下文，边=关系类型+集成模式，标 ACL 落点 |
| 核心保护核对 | 每个核心上下文的上游隔离方式 |

> **返回格式自检**：每条跨界关系都定了权力类型与集成模式；每个核心上下文对外都有 ACL 或等价隔离；每条契约有且仅有一个 owner；每条交互标了同步/异步与强/最终一致；用了 Shared Kernel 的已论证耦合可控。
