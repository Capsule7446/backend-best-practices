# Domain Interactions（领域协作的接口语义化）

## 做什么

定义聚合怎样协作把事做完：哪些事实以**领域事件**广播、哪些跨聚合的**纯业务判断**放进**领域服务**、聚合怎样被**仓储**取存、复杂构造交给**工厂**。全部产出**语言中立契约**（签名 + 语义），不写实现、不绑语言。

职责分界：跨聚合的**纯业务判断**（无 I/O 的规则、资格、计算）→ 领域服务；涉及**加载、保存、事务、外部调用、重试、事件投递**的用例编排 → 应用层（不在本能力产出）；**跨时间的长流程** → 流程管理器（同样不在本能力产出）。本能力遇到编排/长流程诉求时只登记为"应用层待办"，不落成领域服务。

## 需要什么参数

- **必需**：一份含 {聚合目录, 不变量表, 事务与一致性策略, 契约所有权} 的工件。
- **可选**：性能/查询场景、外部系统接口。

## 怎么做

1. **定义领域事件契约**：把"最终一致"的跨聚合协作落成具名事件——名字（过去式）、不可变载荷（**只含订阅方所需、不泄露发布聚合私有内部**）、发布聚合与时机、owner 上下文。
2. **设计领域服务**：把"不属任何单一聚合、需跨聚合做**纯业务判断**"的逻辑定义为无状态领域服务接口（无仓储调度、无事务、无重试、无消息发送——那些属于应用层编排）；带 I/O 的协调诉求登记为"应用层待办"。
3. **设计仓储接口**：每个聚合根一个仓储端口，以聚合为单位取存、不暴露内部实体独立 CRUD，查询方法标查询边界。
4. **设计工厂**：仅当构造有复杂不变量或多步装配时才引入。
5. **为每个契约写语义注释**：前置/后置、触发的不变量编号、事务边界、失败/空值语义。**语义比签名更重要**。
6. **标接口归属层**：领域端口 or 应用编排，标依赖方向（领域只依赖抽象）。
7. **自检无语言泄露**：无 `List<>`/`Promise`/注解，只用中立"类型名 + 基数 + 可空性"。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 领域事件目录 | 事件名、不可变载荷、发布聚合、时机、owner、订阅方 |
| 领域服务接口 | 服务名、中立签名、协调聚合、对应不变量、前置/后置（仅纯业务判断）|
| 应用层待办 | 带 I/O/事务/跨时间的协调诉求：描述 + 涉及聚合 + 一致性要求 |
| 仓储接口 | 每聚合根一份：签名 + 查询边界 + 事务语义；标"领域层端口" |
| 工厂接口（按需）| 复杂构造：签名 + 装配的不变量 |
| 契约语义表 | 每方法：前置/后置、不变量引用、失败与空值语义 |
| 层与依赖标注 | 层归属 + 依赖方向 |

> **返回格式自检**：每个事件载荷不含发布聚合私有内部；每个聚合根恰好一个仓储接口、不暴露内部 CRUD；领域服务只含跨聚合纯业务判断（无仓储调度/事务/重试/消息发送——出现即改登记为应用层待办）；每个方法都有契约语义（不止签名）；全部契约语言中立、无具体语言类型/框架注解；仓储与领域服务都归领域层、依赖只向内。
