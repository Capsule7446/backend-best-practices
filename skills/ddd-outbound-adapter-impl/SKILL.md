# Outbound Adapter Impl（出站适配器实现与装配）

## 做什么

在出站端口背后实现全部技术细节：持久化、事务、事件投递、外部系统调用、读存储——并在最外层把一切装配起来。**核心声明端口，此处兑现端口**；技术选型可换，端口契约不动。

## 需要什么参数

- **必需**：
  - 一份含 {出站端口契约}（仓储/网关/Query/发布端口的签名与契约注释）的工件。
  - 一份含 {事务与事件投递要求}（一致性策略、投递语义、幂等要求、乐观锁约定）的工件。
  - **语言剖面**——目标语言的实现构造与 DI 方式映射。
- **可选**：目标数据库/消息中间件/DI 容器选型、既有连接与迁移设施。

## 怎么做

1. **实现仓储适配器**：
   - 领域模型 ↔ 持久化模型显式映射，**禁止把 DB 行当聚合**。
   - 乐观锁：版本列 + 期望版本比对，不符返回并发冲突（具名，不吞成功）。
   - 维护 **1 聚合 / 1 事务**：单次保存只写一个聚合的边界。
2. **实现 Unit of Work**：聚合保存与 Outbox 写入在**同一事务**内原子提交；回滚即全无。
3. **实现 Outbox 写入与投递器**：事务内 append 集成事件行；独立投递器（轮询/CDC）按序外发，至少一次 + 失败退避重试 + 投递状态可查。
4. **实现 Inbox 去重**：消费侧按消息 ID 判重落库，重复投递只生效一次。
5. **实现外部系统网关（ACL）**：把外部模型翻译成领域语言，外部概念**止步于网关**；超时/重试/熔断封在网关内，不外泄给调用方。
6. **实现 Read Store 适配**：兑现查询端口，可绕聚合直读读模型，返回时标注 freshness。
7. **Composition Root 装配**：在最外层（入口/启动层）new 全部具体实现并注入端口——**核心永不 new 具体实现**。
8. **依赖方向静态自检**：核对领域/应用包不 import 任何出站适配器包；依赖只向内。

## 约束（铁律）

- 每个适配器只兑现自己的端口，不越权调用别的适配器实现。
- 事务边界在此层兑现，但**由谁开启由调用方决定**——适配器不偷开事务。
- 外部系统的类型、错误码、术语一律不穿透 ACL。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 仓储适配器 | 兑现仓储端口；显式映射；乐观锁；1 聚合/1 事务 |
| UoW 实现 | 聚合保存 + Outbox append 同事务原子提交 |
| Outbox 投递器 / Inbox 去重 | 至少一次投递 + 退避重试；消费侧按消息 ID 幂等 |
| 外部系统网关(ACL) | 外部模型 → 领域语言翻译表；超时/重试封装 |
| Read Store 适配 | 兑现查询端口；读模型直读 + freshness 标注 |
| Composition Root 装配 | 装配清单（端口 → 实现 → 注入点），全部在最外层 |
| 依赖方向核对结果 | 领域/应用包 import 核对：通过/不通过 + 违规项 |

> **返回格式自检**：每个出站端口有且仅有对应适配器兑现且签名逐条一致；仓储含显式映射与乐观锁、维护 1 聚合/1 事务；聚合保存与 Outbox 写入同事务（可指认代码位置）；外部概念零穿透（ACL 内完成翻译）；全部具体实现仅在 Composition Root 实例化、核心零 new；领域/应用包不 import 任何适配器包（静态可验证）。
