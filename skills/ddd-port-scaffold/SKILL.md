# Port Scaffold（接口骨架）

## 做什么

把端口契约翻译成目标语言的接口构造——先立约、不填肉（实现由 adapter-impl 接手）。适配任意面向接口开发的语言。

## 需要什么参数

- **必需**：端口契约规范；聚合目录与不变量表；`语言剖面`入参——目标语言标识（`java|go|ts|python|csharp|rust|kotlin|...`）。
- **可选**：既有代码风格约定（命名、包结构）、DI 方式偏好。

## 怎么做

1. **载入语言剖面**：从 `references/language-profiles.md` 取该语言的构造映射（接口/值对象/不可变/相等性/DI）；未收录则按"剖面问卷"现场采集 5 项。
2. **映射端口**：每个中立端口 → 该语言的接口/trait/Protocol，逐方法翻译签名，**保留契约注释**（前置/查询边界/事务/不变量引用）。
3. **映射值对象与标识**：值对象 → 不可变值类型（按值相等）；标识 → ID 类型。
4. **生成依赖方向骨架**：领域包只声明接口与领域类型；基础设施包留空占位。确保依赖只向内。
5. **植入门禁注释**：每个接口/方法注明对应不变量编号与一致性策略。
6. **输出骨架文件清单**（仅签名 + 注释，无逻辑），标"待实现"占位点。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 端口接口骨架 | 目标语言接口文件：签名 + 契约注释，无实现 |
| 值对象/标识骨架 | 不可变值类型与 ID 类型，含相等性策略 |
| 依赖方向说明 | 包/模块结构 + 依赖指向（内核零外部依赖）|
| 待实现清单 | 文件、占位点、对应聚合/不变量 |
| 所用语言剖面 | 采用的剖面与五项构造映射（可复现）|

> **返回格式自检**：每个端口方法保留契约注释；值对象用该语言不可变构造且按值相等；领域内核无基础设施/框架依赖；骨架内**无业务实现逻辑**；每个接口/方法关联到具体不变量编号；未收录语言已补齐剖面五项。
