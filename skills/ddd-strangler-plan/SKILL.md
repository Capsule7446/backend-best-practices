# Strangler Plan（绞杀者迁移计划）

## 做什么

让新实现像绞杀榕一样在旧系统外围逐步长出、逐步接管，直到旧实现枯萎。把接缝转化为**有序、可回滚的施工序列**：每片独立交付、先有特征化测试兜底、经 ACL 隔离、灰度接管、随时可回滚。

## 需要什么参数

- **必需**：一份含 {接缝清单, ACL 落点, 切割风险评估, 特征化测试需求, 行为黑盒清单} 的工件。
- **可选**：发布/灰度能力（特性开关、路由、影子流量）、回滚基础设施、业务低峰窗口。

## 怎么做

1. **切片化**：按接缝切成最小可独立交付切片（一片 = 一条缝 + 其 ACL + 一段可验收行为）。优先级 = 价值高 x 风险低 x 依赖少。
2. **排迁移批次**：先易后难、先低耦合后高耦合，先用低风险切片验证整套绞杀机制。
3. **设计流量切换策略**：特性开关 / 按比例路由 / 影子流量比对；明确接管前观察指标。
4. **设计回滚策略**：每片定义回滚触发信号 + 一键回退 + 数据不脏保证。
5. **圈定特征化测试范围**：每片切割前必补的特征化测试。
6. **定义切片 DoD**：特征化测试过 + 新实现验收过 + 流量切稳 + 旧路径可回滚。
7. **规划旧代码退役**：每片旧实现/旧字段/临时 ACL 的下线时机。
8. **输出迁移路线图**：批次顺序 + 每片的切换/回滚/测试/DoD + 里程碑。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 切片清单 | 切片、对应接缝/ACL、可验收行为、价值/风险/依赖评分 |
| 迁移批次顺序 | 有序批次 + 排序理由 |
| 流量切换策略 | 每片：灰度方式 + 接管前观察指标 |
| 回滚策略 | 每片：回滚触发 + 一键回退 + 数据安全 |
| 特征化测试范围 | 每片切割前必补的特征化测试 |
| 切片 DoD | 每片完成判据 |
| 旧代码退役计划 | 每片旧实现/旧字段/临时 ACL 下线时机 |
| 迁移路线图 | 批次序列 + 里程碑总览 |

> **返回格式自检**：每个切片可独立交付、独立回滚；批次顺序遵循"先易后难"；每片有流量切换方式与接管前观察指标；每片有回滚触发与一键回退且数据不脏；每片切割前特征化测试范围已圈定（无裸切）；每片有 DoD；临时 ACL/旧字段都有退役时机。
