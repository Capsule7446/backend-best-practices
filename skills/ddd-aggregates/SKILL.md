# Aggregates（聚合设计）

## 做什么

按"必须在同一事务里保持为真的不变量"划聚合（而非按数据相关）。聚合根是这组不变量的唯一守门人；一个事务只改一个聚合实例；跨聚合一致性走最终一致。

## 需要什么参数

- **必需**：一份含 {目标上下文词汇表, 策略, 跨界一致性语义} 的工件。
- **可选**：性能/并发要求、预期数据规模、现有数据模型（改造切片时）。

## 怎么做

1. **先列不变量，不画对象**：把"任何时刻都必须为真"的规则逐条写出。不变量是聚合设计的唯一第一性依据。
2. **给每条不变量定作用域**：强制它需要哪些数据"同一瞬间一致"——这组数据是一个聚合候选边界。
3. **圈聚合、选聚合根**：把强制同一组不变量的实体/值对象圈进一个聚合；指定唯一根作为对外入口与守门人。
4. **区分实体与值对象**：优先值对象（不可变、好测、无别名）。
5. **定事务边界**：一个事务只创建/修改一个聚合实例；需协调多聚合改用领域事件 + 最终一致。
6. **设计跨聚合一致性**：强一致（多半说明边界划错）vs 最终一致（用事件）。
7. **聚合大小体检**：过大（数据袋、加载慢、并发锁）拆小；过小到不保护任何不变量 → 可能只是值对象。
8. **用聚合引用而非对象引用**：聚合间以 ID 互引。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 不变量表 | 编号(INV-n)、规则、作用域、强制时机、所属聚合 |
| 聚合目录 | 聚合名、聚合根、内含实体/值对象、对外根方法 |
| 值对象清单 | 值对象、属性、不可变性与相等性 |
| 事务边界声明 | 每聚合：一事务可改范围、被禁跨聚合写 |
| 一致性策略表 | 跨聚合关系、强/最终、事件、补偿 |
| 聚合引用图 | 聚合间仅以 ID 互引 |

> **返回格式自检**：每个聚合至少强制一条具名不变量；每条不变量归属唯一聚合且能单事务内强制；一个事务只改一个聚合、无跨聚合写；聚合间只用 ID 引用；跨聚合一致性逐条标强/最终、最终一致项都指定事件与补偿；无"大数据袋"聚合。
