# Application Impl（应用层实现）

## 做什么

在接口骨架背后实现应用层用例：把授权、幂等、事务、聚合调用、事件外发编排成**固定形状**的 Handler。应用层只做编排与翻译——**业务判断全部委托聚合与领域服务，技术细节全部委托端口**。

## 需要什么参数

- **必需**：
  - 一份含 {应用用例目录, Input Port 清单, Command·Result 契约, 授权分层表, 错误语义表} 的工件。
  - 一份含 {编排设计}（用例步骤、事务边界、领域事件→集成事件映射）的工件。
  - 一份含 {接口骨架, 待实现清单} 的工件；**语言剖面**。
- **可选**：视图契约（无独立读模型的视图，其查询字段/权限/分页约束——Query Handler 与 View DTO 按此实现）；项目 DI/包结构约定、幂等存储约定。

## 怎么做

1. **实现 Command Handler（固定形状，逐用例套用）**：
   ① 用例级授权（Unauthorized 即止）→ ② 幂等查取/占用（命中即返回既有结果）→ ③ 开启 UoW →
   ④ 经仓储端口加载聚合（不存在 → NotFound）→ ⑤ 调用聚合方法/领域服务（业务判定在里面）→
   ⑥ 保存聚合（带期望版本）→ ⑦ 领域事件映射为集成事件并 append Outbox →
   ⑧ 提交 UoW → ⑨ 存幂等结果 → ⑩ 映射 Result（只含本事务已确认事实）。
   步骤可按用例契约裁剪（如无幂等要求），但裁剪必须显式标注豁免理由。
2. **实现 Query Handler**：查询授权约束**下推**为过滤条件传给 Query Port（不是取回后再筛）→ 调用 Query Port → 返回 View DTO + freshness 标注。
3. **实现应用事件处理器**：订阅集成/应用事件 → 按消息幂等键去重 → 翻译成对应用例调用（复用 Command Handler 形状），不在处理器里写业务分支。
4. **落实错误映射**：领域具名拒绝 / 乐观锁冲突 / 幂等冲突 → 具名应用错误（沿用错误语义表命名），逐条登记映射表，不吞错、不改名。
5. **依赖与分支合规自检**：import 只指向端口抽象与领域类型；逐 Handler 核对分支类型（见铁律）。

## 约束（铁律）

- **只依赖端口抽象**：不 import 任何具体适配器、驱动、框架。
- **允许的分支只有两类**：技术结果（NotFound / 授权拒绝 / UoW 失败 / 乐观锁冲突 / 幂等冲突）与领域具名结果（聚合返回的具名拒绝）。
- **禁止** `if aggregate.status == ...` 式业务判断；禁止直接读改实体字段绕过聚合方法——想加分支先回聚合加方法。
- **外部网络调用不进无界事务**：要么在事务外，要么经 Outbox 异步化。
- 同幂等键不同参数视为幂等冲突，返回具名错误而非旧结果。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| Command Handler 实现 | 每用例一个，严格套用①—⑩固定形状；缺步骤须标注豁免理由 |
| Query Handler 实现 | 授权约束下推 + Query Port 调用 + View DTO + freshness |
| 应用事件处理器 | 订阅 + 去重 + 转用例调用；无业务分支 |
| 错误映射表 | 领域拒绝/并发冲突/幂等冲突 → 具名应用错误，逐条对应 |
| 合规核对结果 | import 核对 + 分支类型核对：通过/不通过 + 违规项 |

> **返回格式自检**：应用层 import 仅端口抽象与领域类型（静态可验证）；每个 Command Handler 覆盖固定形状十步或标注豁免；所有分支属于技术结果或领域具名结果两类，无业务状态判断、无实体字段直改；错误映射与错误语义表逐条对应、无吞错；外部网络调用不在无界事务内；Result 只含本事务已确认事实。
