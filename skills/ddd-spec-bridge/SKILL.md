# Spec Bridge（规范衔接：完整后端契约）

## 做什么

把散落在多个工件里的模型，收敛成**一份单一、自洽、可被任意面向接口语言直接消费的后端契约规范**——覆盖 Domain、Application、读侧与可靠性语义。硬纪律：**零语言绑定**。

## 需要什么参数

- **必需**：含 {不变量表, 聚合目录, 事件/服务/仓储/工厂契约} 的战术工件；含 {用例目录, 编排设计} 的应用工件。
- **可选**：流程管理器设计；视图契约/读模型设计（有读侧时）；契约所有权；目标语言（仅作记录）。

## 怎么做

1. **汇总 Domain 端口**：仓储/领域服务/工厂接口收成端口总表，每个含用途、所属聚合/上下文、全部方法。
2. **规范化方法签名（中立类型）**：`方法名(参数: 中立类型) -> 返回类型 | null`，标基数与可空性。禁语言特有类型。
3. **固化契约语义**：每方法附前置/后置、事务/查询边界、所守不变量编号、失败/空值语义。
4. **固化应用用例契约**：每个用例的 Input Port、Command/Result 形状、授权分层、错误语义；每个查询用例的 Query Port 与 View DTO 契约。
5. **汇总出站端口**：仓储之外的 UoW、Outbox、幂等存储、外部网关、Read Store 端口与语义。
6. **冻结事件 schema**：领域事件与集成事件分列——每事件给不可变载荷、类型、版本、owner、向后兼容策略，标注领域→集成映射归属。
7. **固化可靠性语义**：每写用例的事务边界、幂等作用域与崩溃窗口策略、事件投递策略、重试边界（从编排工件转录为规范条目）。
8. **沉淀不变量为可验收命题**：每条 INV 改写成"当 X 时保证 Y"的可观测断言。
9. **写验收准则**：关键用例（走应用用例路径）、不变量、读侧（有则含新鲜度/权限）给 Given-When-Then。
10. **（可选）导出 OpenSpec 变更集**；整份规范打版本号，标跨上下文契约 owner。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| Domain 端口规范 | 端口总表：中立签名 + 完整契约语义 |
| 应用用例契约 | 每用例：Input Port、Command/Result、授权分层、错误语义 |
| 出站端口规范 | UoW/Outbox/幂等存储/外部网关/Read Store：签名 + 语义 |
| 事件 schema | 领域/集成分列：载荷+类型+版本+owner+兼容策略+映射归属 |
| 读侧契约（可选）| Query Port、View DTO、权限约束、新鲜度语义 |
| 可靠性语义 | 每写用例：事务边界、幂等、投递策略、重试边界 |
| 不变量命题集 | 每条 INV → 可观测断言 |
| 验收准则 | 用例/不变量/读侧的 Given-When-Then |
| 规范元信息 | 版本号、覆盖上下文、契约所有权、生成日期 |
| （可选）OpenSpec 变更集 | 符合 OpenSpec 的规范提案 |

> **返回格式自检**：规范中**零语言特有类型**；每个端口方法带完整契约语义；每个写用例有 Input Port、错误语义与可靠性条目；Command/Result/View DTO 不引用领域实体内部结构；每条不变量已改写为可观测断言；每个事件 schema 有版本与 owner；关键用例/不变量都有 Given-When-Then；规范自洽（引用的类型/事件/端口都在内定义）。
