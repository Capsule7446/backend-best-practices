# Spec Bridge（规范衔接：语言中立接口契约）

## 做什么

把散落在多个工件里的模型，收敛成**一份单一、自洽、可被任意面向接口语言直接消费的接口契约规范**。硬纪律：**零语言绑定**——只有端口、签名、语义、不变量、验收准则。

## 需要什么参数

- **必需**：一份含 {不变量表, 聚合目录, 事件/服务/仓储/工厂契约} 的战术工件。
- **可选**：契约所有权（用于跨上下文事件 schema 的版本策略）；目标语言（仅作记录，不影响规范内容）。

## 怎么做

1. **汇总端口清单**：仓储/领域服务/工厂接口收成一张端口总表，每个含用途、所属聚合/上下文、全部方法。
2. **规范化方法签名（中立类型）**：`方法名(参数: 中立类型) -> 返回类型 | null`，标基数与可空性。禁语言特有类型。
3. **固化契约语义**：每方法附前置/后置、事务/查询边界、所守不变量编号、失败/空值语义。
4. **冻结领域事件 schema**：每事件给不可变载荷、类型、版本、owner、向后兼容策略。
5. **沉淀不变量为可验收命题**：每条 INV 改写成"当 X 时保证 Y"的可观测断言。
6. **写验收准则**：关键用例与不变量给 Given-When-Then。
7. **（可选）导出 OpenSpec 变更集**。
8. **版本与所有权标注**：整份规范打版本号，标每个跨上下文契约 owner。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 端口契约规范 | 端口总表：中立签名 + 完整契约语义 |
| 领域事件 schema | 载荷字段+类型+版本+owner+兼容策略 |
| 不变量命题集 | 每条 INV → 可观测断言 |
| 验收准则 | 关键用例/不变量的 Given-When-Then |
| 规范元信息 | 版本号、覆盖上下文、契约所有权、生成日期 |
| （可选）OpenSpec 变更集 | 符合 OpenSpec 的规范提案 |

> **返回格式自检**：规范中**零语言特有类型**；每个端口方法带完整契约语义；每条不变量已改写为可观测断言；每个事件 schema 有版本与 owner；关键用例/不变量都有 Given-When-Then；规范自洽（引用的类型/事件都在内定义）。
