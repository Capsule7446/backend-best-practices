# Adapter Impl（接口背后的实现）

> **已废弃**：该能力已按层拆分为 `ddd-domain-impl` / `ddd-application-impl` / `ddd-inbound-adapter-impl` / `ddd-outbound-adapter-impl` 四个实现 Skill。本文件仅作兼容入口保留，计划 **0.3.0 移除**——新流程请直接使用按层拆分后的 Skill。

## 做什么

在已生成的接口骨架背后写真实代码：**领域核心只实现领域逻辑、只依赖端口抽象；一切技术细节封在实现端口的适配器里**。换语言 = 换骨架 + 换适配器，领域逻辑形状不变。

## 需要什么参数

- **必需**：接口骨架与待实现清单；契约语义与不变量命题；所用语言剖面。
- **可选**：项目现有工程规范（包结构、测试框架、DI）、目标数据库/消息中间件。

## 怎么做

1. **实现值对象与标识**：构造校验、按值相等。
2. **实现聚合根行为**：每个根方法**在内部强制其不变量**，非法跃迁直接拒绝（抛中立领域错误 / 返回 Result）。
3. **实现领域服务**：跨聚合纯业务判定/编排，仅依赖端口抽象。
4. **实现仓储适配器**：领域模型与持久化模型解耦（做映射，禁止把 DB 行当聚合），维护"1 聚合/1 事务"。
5. **实现外部系统适配器（ACL）**：把外部概念翻译成领域语言，隔离在防腐层，绝不渗进核心。
6. **接事件发布/订阅**：落实跨聚合最终一致与补偿逻辑。
7. **装配依赖（DI）**：在最外层把适配器注入领域端口——核心永不 new 具体实现。
8. **依赖方向自检**：静态核对领域核心未 import 任何基础设施/框架。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 聚合/值对象实现 | 根方法内强制不变量；值对象不可变按值相等；标 INV 编号 |
| 领域服务实现 | 仅依赖端口；无基础设施引用 |
| 仓储适配器 | 实现仓储端口；领域↔持久化映射；维护事务边界 |
| 外部系统适配器(ACL) | 外部模型翻译为领域语言；隔离在防腐层 |
| 事件发布/订阅实现 | 发布点 + 处理器 + 补偿逻辑 |
| 依赖装配 | 外层 DI 组装；依赖方向核对结果 |

> **返回格式自检**：领域核心零基础设施/框架依赖（静态可验证）；每个根方法内部强制其不变量；仓储维护"1 聚合/1 事务"且领域与持久化解耦；外部系统一律经 ACL；依赖在最外层装配、核心从不实例化具体实现；实现与契约语义逐条对应、无擅自加方法。
