# Inbound Adapter Impl（入站适配器实现）

## 做什么

把每个入口（HTTP / 消息 / 定时 / CLI）实现为**协议世界到用例世界的薄翻译层**：解析协议、校验格式、认证来者、调用 Input Port、把结果与具名错误翻译回协议表达。入站层只做翻译，**一条业务规则都不写**。

## 需要什么参数

- **必需**：
  - 一份含 {入口清单} 的工件（每个入口的协议类型、路由/主题/排程、对应用例）。
  - 一份含 {Input Port 清单, Command·Result 契约, 错误语义表} 的工件。
  - **语言剖面**——目标语言/框架的协议构造映射。
- **可选**：Web/消息框架选型、既有认证中间件、API 风格约定（REST/RPC）。

## 怎么做

1. **逐入口映射**：每个入口绑定唯一 Input Port 调用；建立请求字段 → Command/Query DTO 字段的映射表，不多传、不少传。
2. **协议解析与格式校验**：缺字段、类型错、超长、格式非法在此层直接拒绝（协议级 4xx / 消息进死信）；**业务校验不做**——那是聚合的事。
   校验只到"形状合法"为止，语义合法性交给领域判定。
3. **接入 Authentication**：认证（"你是谁"）在此层完成，产出已认证主体随命令传入；授权（"你能不能"）留给应用层，此层不判。
4. **协议级错误映射**：具名应用错误 → 协议表达，逐条登记：
   - HTTP：状态码 + 问题详情（不泄内部细节）。
   - 消息：ACK / NACK 重试 / 死信的选择依据。
   - CLI/Job：退出码与日志。
5. **Consumer 去重钩子**：对重复投递提供去重钩子——从消息中提取幂等键传入用例；入站层自身不建业务状态。
6. **薄度自检**：静态核对入站层 import——只指向 Input Port、DTO 与协议库；无仓储、无聚合操作、无领域服务。

## 约束（铁律）

- **不含业务规则**：任何"能不能取消/该不该通过"的判断都不出现在此层。
- **禁止直连 Repository、禁止直接修改聚合**——进入系统的唯一通道是 Input Port。
- 一个入口只翻译、只调用、只回译；出现第二职责即拆分。
- 错误回译不泄露堆栈与内部标识，对外只讲协议语言。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| Controller/Consumer/Job/CLI 实现 | 每入口一个薄适配：解析 → 认证 → 调 Input Port → 回译 |
| 请求映射表 | 入口字段 ↔ Command/Query DTO 字段，逐条对应 |
| 认证接入 | 认证方式 + 已认证主体的传递形状；授权归属标注（应用层） |
| 协议级错误映射表 | 具名应用错误 → HTTP 状态/消息 ACK·NACK·死信/退出码 |
| 去重钩子 | 幂等键提取规则 + 传入位置 |
| 薄度核对结果 | 入站层 import 核对：通过/不通过 + 违规项 |

> **返回格式自检**：每个入口唯一对应一个 Input Port 调用且有字段映射表；格式校验与业务校验分离（此层只拒格式）；认证在此层、授权不在此层；错误语义表中每个具名错误都有协议映射、无遗漏无吞错；Consumer 有去重钩子；入站层零仓储/聚合/领域服务引用（静态可验证）。
