# Domain Impl（领域层实现）

## 做什么

在接口骨架背后实现领域层：值对象、聚合根、领域服务与领域事件。**只写业务规则，不碰任何技术细节**——事务、重试、消息发送、持久化一概不属于此层。换语言 = 换骨架，领域逻辑形状不变。

## 需要什么参数

- **必需**：
  - 一份含 {接口骨架, 待实现清单} 的工件（值对象/聚合/领域服务的签名与占位点）。
  - 一份含 {契约语义, 不变量命题} 的工件（每条不变量带 INV 编号、非法跃迁的具名拒绝语义、预期领域事件）。
  - **语言剖面**——目标语言的不可变构造/相等性/错误表达映射。
- **可选**：项目命名与包结构约定、既有测试框架。

## 怎么做

1. **实现值对象与标识**：
   - 构造即校验——非法值拒于构造点，构造成功即合法。
   - 按值相等；不可变（"修改" = 产生新值）。
2. **实现聚合根行为**：
   - 每个根方法**在内部强制其不变量**，代码处标注所守 INV 编号。
   - 非法状态跃迁不抛技术异常，**返回具名领域拒绝**（中立领域错误 / Result），名字取自契约语义。
   - 内部状态只经根方法变更，不暴露可绕过不变量的 setter。
3. **实现领域服务**：仅当判定跨越多个聚合且不属于任何一个时才建；**纯业务判定**——输入输出皆领域类型，无 I/O、无副作用。
4. **定义领域事件**：
   - 过去式命名，携带聚合标识与事实数据。
   - 在聚合方法内**登记发布点**（记录"发生了什么"待外层取走，不做任何发送）。
5. **依赖纯净性自检**：静态核对领域包 import——零基础设施、零框架、零协议库；有违规即修复后重查。

## 约束（铁律）

- 领域层**零基础设施/框架依赖**，且必须静态可验证（看 import 清单即可判定）。
- **不含**事务控制、重试、消息发送、持久化细节。
- 不擅自增加契约之外的公开方法；实现与契约语义逐条对应。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 值对象/标识实现 | 构造校验 + 按值相等 + 不可变；非法值拒于构造点 |
| 聚合根实现 | 每个根方法标注所守 INV 编号；非法跃迁返回具名领域拒绝 |
| 领域服务实现 | 跨聚合纯判定；输入输出皆领域类型；无 I/O |
| 领域事件定义与发布点登记 | 过去式事件 + 聚合标识 + 事实数据；发布点登记表（根方法 → 事件） |
| 依赖纯净性核对结果 | 领域包 import 核对：通过/不通过 + 违规项清单 |

> **返回格式自检**：领域包零基础设施/框架依赖（静态可验证）；每个根方法内部强制其不变量并标 INV 编号；所有非法跃迁返回具名领域拒绝而非技术异常；值对象不可变且按值相等；领域服务无 I/O 无副作用；事件皆过去式且登记了发布点；无契约之外的擅自方法。
