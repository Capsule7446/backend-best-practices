# Seam Finder（接缝与防腐层识别）

## 做什么

在现状模型上定位"能在不改动其行为的前提下改变其行为"的接缝（Michael Feathers），并指定防腐层（ACL）落点，把要剥离的新模型与残留旧代码隔开。产出"在哪下刀、用什么隔离、风险多大"。

## 需要什么参数

- **必需**：一份含 {现状模型, 隐含上下文, 坏味道清单, 行为黑盒清单} 的勘测工件。
- **可选**：调用/依赖图、数据库外键关系、运行时调用频次。

## 怎么做

1. **沿隐含上下文边界找缝**：语言断裂处优先下刀。
2. **盘点可用接缝类型**：已有接口、可插接口的调用点、可改为事件的同步调用、模块/包边界、数据库视图层。
3. **定位防腐层落点**：在"新剥离模型"与"残留旧系统"之间确定 ACL 物理落点，把旧脏概念翻译成干净领域语言。
4. **评估每条缝的切割成本与风险**：数据耦合、事务耦合、调用频次、隐藏副作用；打"易切/难切/暂不可切"。
5. **处理同名异义字段**：规划在接缝处拆分/翻译。
6. **标特征化测试需求**：每条缝切割前需锁住的跨缝行为。
7. **输出接缝地图**：把缝、ACL 落点、风险等级画成可决策的地图。

## 返回什么

| 工件 | 结构 |
| :--- | :--- |
| 接缝清单 | 缝位置、类型（接口/事件/模块/视图）、连接两侧、切割难度 |
| 防腐层落点 | ACL 名、隔离的新旧两侧、翻译的概念、读/写方向 |
| 切割风险评估 | 每条缝的数据/事务/频次/副作用风险 + 综合等级 |
| 同名异义拆分方案 | 混用字段 -> 在缝处如何拆分/翻译 |
| 特征化测试需求 | 每条缝切割前需锁住的跨缝行为 |
| 接缝地图 | 现状模型上标所有缝、ACL、风险等级 |

> **返回格式自检**：每个隐含上下文边界都评估过是否为可用接缝；每条要切的缝指定了类型与切割难度；新旧之间都规划了 ACL 且方向明确；数据/事务耦合风险逐缝评估、"暂不可切"已标原因；同名异义拆分方案已给出；每条缝的特征化测试需求已登记。
