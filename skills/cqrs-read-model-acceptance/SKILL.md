# Read Model Acceptance（读侧验收）

## 做什么

验"读侧实现是否忠于查询契约、权限是否恒隔离、投影在故障与乱序下是否收敛"。每个验收项必须落成可执行测试并留下证据，不接受口头"已覆盖"。

## 需要什么参数

- **必需**：查询契约与字段映射；读侧实现（查询入口、投影处理器、checkpoint）；权限约束（租户/行级/字段级）。
- **可选**：新鲜度/延迟容忍指标、查询性能预算、consistency token 机制、降级回退策略、Retry/DLQ 配置。

## 怎么做

1. **查询契约测试**：每个契约覆盖筛选、排序、分页（首页/中间页/末页/越界）、空结果。
2. **字段正确性测试**：字段来源与转换规则逐条比对真源，冗余字段与真源一致。
3. **权限隔离测试**：租户/行级/字段级各写正例 + **越权负例**——他租户/无权者查询后零泄漏，断言范围含 total 计数、分页游标、聚合值、缓存命中。
4. **新鲜度断言**：响应携带 as_of/generated_at；Projection Lag 超过容忍阈值时 stale 标记正确。
5. **投影收敛测试**：注入重复事件、乱序事件、全量重放，读模型终态一致（幂等收敛）。
6. **Retry/DLQ 行为测试**：投影失败按策略重试；超限进 DLQ 且不阻塞后续事件。
7. **Checkpoint 重建等价性**：清空读存储，从 checkpoint/事件流重建，结果与增量投影逐行逐字段等价。
8. **read-your-writes 测试**：走 consistency token 路径，写后立读能看到自己的写入。
9. **性能预算测试**：关键查询在目标数据量与并发下满足延迟/吞吐预算。
10. **降级与回退测试**：读存储不可用时按既定行为执行（回退真源 / 返回 stale / 明确失败），不静默返回空结果。
11. **汇总验收报告**：每项给 pass/fail + 证据（测试文件名 + 用例名）。

## 返回什么

验收测试集分类表：

| 验收类别 | 覆盖项 |
| :--- | :--- |
| 查询契约 | 筛选 / 排序 / 分页 / 空结果 |
| 字段正确性 | 来源比对 / 转换规则 / 冗余一致性 |
| 权限隔离 | 租户 / 行级 / 字段级，各含越权负例 |
| 新鲜度 | as_of 存在 / lag 阈值 / stale 标记 |
| 投影收敛 | 重复 / 乱序 / 重放下终态一致 |
| 失败处理 | Retry 策略 / DLQ / 不阻塞后续事件 |
| 重建等价 | checkpoint 重建 ≡ 增量投影结果 |
| read-your-writes | consistency token 路径 |
| 性能 | 查询延迟 / 吞吐预算 |
| 降级回退 | 读存储不可用时的既定行为 |

验收报告结构（每个验收项一行）：

| 字段 | 含义 |
| :--- | :--- |
| category | 验收类别（上表之一） |
| check | 验收点描述 |
| result | pass 或 fail |
| evidence | 测试文件名 + 用例名（必填） |
| notes | 失败原因 / 阻断说明（可选） |

> **返回格式自检**：每个验收项都有可执行测试与证据字段（测试文件名 + 用例名）；仅写"已覆盖"而无测试名视为无证据，不得计入通过；越权负例存在且断言零泄漏（含 total/游标/聚合/缓存）；重建等价性给出逐行逐字段比对结果；报告对每项给出明确 pass/fail 结论。
